# 3.2 — Text-to-Speech Basics

## Goal

Add a `generate_speech` tool that converts text (e.g. a question prompt) to an MP3 audio file, saves it to the `audios/` folder, and traces the TTS API call in LangSmith so you can monitor cost and latency.

---

## Step 1 — Choose your TTS provider

This guide uses the **OpenAI TTS API** (no extra package — already available via `openai`).  
An ElevenLabs alternative is noted at the end.

---

## Step 2 — Update `.env.example`

```env
# TTS settings (OpenAI)
TTS_MODEL=tts-1
TTS_VOICE=alloy
```

Available voices: `alloy`, `echo`, `fable`, `onyx`, `nova`, `shimmer`  
Available models: `tts-1` (fast, lower cost), `tts-1-hd` (higher quality)

Copy to `.env` and set values.

---

## Step 3 — Create `app/tools/tts_tool.py`

```python
# app/tools/tts_tool.py
import os
from pathlib import Path
from openai import OpenAI
from langsmith import traceable
from dotenv import load_dotenv

load_dotenv()

_client = OpenAI()
_TTS_MODEL = os.getenv("TTS_MODEL", "tts-1")
_TTS_VOICE = os.getenv("TTS_VOICE", "alloy")
_AUDIOS_DIR = Path("audios")


@traceable(name="generate_speech")
def generate_speech(text: str, output_filename: str) -> dict:
    """
    Convert text to speech using the OpenAI TTS API and save to audios/ folder.

    Args:
        text (str): The text to synthesize into speech.
        output_filename (str): Filename (e.g. 'question_01.mp3') to save inside audios/.

    Returns:
        dict with keys: file_path, text_length, model, voice
    """
    _AUDIOS_DIR.mkdir(parents=True, exist_ok=True)
    output_path = _AUDIOS_DIR / output_filename

    print(f"TOOL: Generating speech for {len(text)} chars → {output_path} ...")

    with _client.audio.speech.with_streaming_response.create(
        model=_TTS_MODEL,
        voice=_TTS_VOICE,
        input=text,
        response_format="mp3",
    ) as response:
        response.stream_to_file(output_path)

    file_size_kb = round(output_path.stat().st_size / 1024, 1)
    print(f"TOOL: Audio saved — {file_size_kb} KB at {output_path}")

    return {
        "file_path": str(output_path),
        "text_length": len(text),
        "model": _TTS_MODEL,
        "voice": _TTS_VOICE,
        "size_kb": file_size_kb,
    }
```

### Key points

| Detail                                           | Reason                                                                   |
| ------------------------------------------------ | ------------------------------------------------------------------------ |
| `with_streaming_response`                        | Streams bytes directly to disk — avoids buffering large audio in memory. |
| `response_format="mp3"`                          | Widely compatible; supported by React Native's `expo-av`.                |
| `@traceable`                                     | LangSmith logs text length, model, voice, and file path for each call.   |
| `_AUDIOS_DIR.mkdir(parents=True, exist_ok=True)` | Creates `audios/` automatically if missing.                              |

## Step 4 — Verify the tool standalone

```python
# test_tts.py  (run once, then delete)
from app.tools.tts_tool import generate_speech

result = generate_speech(
    text="Listen carefully and answer the question that follows.",
    output_filename="test_prompt.mp3",
)
print(result)
```

Run:

```bash
uv run python test_tts.py
```

Expected:

```
TOOL: Generating speech for 53 chars → audios/test_prompt.mp3 ...
TOOL: Audio saved — 12.4 KB at audios/test_prompt.mp3
{'file_path': 'audios/test_prompt.mp3', 'text_length': 53, 'model': 'tts-1', 'voice': 'alloy', 'size_kb': 12.4}
```

Open `audios/test_prompt.mp3` in any media player to confirm the audio.

## Step 5 — Monitor TTS cost in LangSmith

1. Open LangSmith → your project.
2. Find the `generate_speech` trace.
3. In **Metadata** you will see `text_length` — multiply by $0.000015 per character (`tts-1`) to estimate cost.
4. For detailed cost tracking, pass `metadata={"chars": len(text)}` inside the function if you want it surfaced as a top-level field.

## Step 6 — Playing audio in a React Native app (reference)

Add `expo-av` to your React Native project:

```bash
npx expo install expo-av
```

Load and play the MP3 served from your FastAPI endpoint:

```typescript
import { Audio } from "expo-av";

async function playQuestion(url: string) {
  const { sound } = await Audio.Sound.createAsync({ uri: url });
  await sound.playAsync();
}
```

The FastAPI endpoint that serves the file is added in section 3.4.

## Optional — ElevenLabs alternative

If you prefer ElevenLabs for higher naturalness:

```bash
# add to pyproject.toml
"elevenlabs>=1.2",
```

```env
# .env.example
ELEVENLABS_API_KEY=your_elevenlabs_key_here
ELEVENLABS_VOICE_ID=your_voice_id_here
```

```python
from elevenlabs.client import ElevenLabs

_el_client = ElevenLabs(api_key=os.getenv("ELEVENLABS_API_KEY"))

@traceable(name="generate_speech_elevenlabs")
def generate_speech_elevenlabs(text: str, output_filename: str) -> dict:
    audio = _el_client.text_to_speech.convert(
        voice_id=os.getenv("ELEVENLABS_VOICE_ID"),
        text=text,
        model_id="eleven_multilingual_v2",
    )
    output_path = _AUDIOS_DIR / output_filename
    with open(output_path, "wb") as f:
        for chunk in audio:
            f.write(chunk)
    return {"file_path": str(output_path)}
```
