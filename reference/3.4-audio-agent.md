# 3.4 — Building the Complete Audio Agent

## Goal

Wire everything built in 3.1–3.3 into a single **Audio Agent** with five tools, expose it through a FastAPI endpoint, and view the full hierarchical trace in LangSmith.

---

```
POST /api/v1/audio/evaluate
        │
        ▼
┌────────────────────────┐
│  AudioRouter           │  app/routes/audio_routes.py
```

## Step 2 — Add the `load_question_audio` tool

This is a thin wrapper that calls `generate_speech` to produce the audio file for a question, then returns its path. Separating it makes the agent's ReAct trace clearer.

```python
# app/tools/audio_question_tool.py
from langsmith import traceable
from .tts_tool import generate_speech


@traceable(name="load_question_audio")
def load_question_audio(question_text: str, question_id: str) -> dict:
    """
    Convert a question into speech and save it to the audios/ folder.

    Args:
        question_text (str): The question or listening prompt to speak aloud.
        question_id (str): Unique identifier used to name the output file.

    Returns:
        dict with keys: file_path, question_id
    """
    filename = f"question_{question_id}.mp3"
    result = generate_speech(text=question_text, output_filename=filename)
    return {"file_path": result["file_path"], "question_id": question_id}
```

```python
AUDIO_SYSTEM_PROMPT = """
You are a listening and pronunciation evaluation agent for an English language learning app.

Given a question, a reference answer, and a user's audio response, you must:
1. Use load_question_audio to generate the question audio (if not already generated).
2. Use transcribe_audio to transcribe the user's audio file to text.
3. Use score_pronunciation to compare the reference answer with the transcription.
4. Use generate_speech to create an audio feedback message for the user.
5. If the exercise type is 'comprehension', use generate_comprehension_passages
   to create a follow-up reading passage and validate the user's answer.

Always return a structured evaluation with: score, transcript, feedback, and any
comprehension questions generated.
""".strip()
```

## Step 4 — Create the response schema and agent

```python
# app/agent/audio_agent.py
from pydantic import BaseModel
from typing import List, Optional
from langchain.agents import create_agent
from dotenv import load_dotenv

from .prompts import AUDIO_SYSTEM_PROMPT
from ..tools.audio_question_tool import load_question_audio
from ..tools.transcription_tool import transcribe_audio
from ..tools.pronunciation_tool import score_pronunciation
from ..tools.tts_tool import generate_speech
from ..tools.comprehension_tool import generate_comprehension_passages

load_dotenv()


class MisPronunciationDetail(BaseModel):
    expected: str
    heard: str
    similarity: int


class AudioEvaluationOutput(BaseModel):
    score: int                                      # 0–100
    transcript: str                                 # Whisper output
    correct_words: List[str]
    mispronounced_words: List[MisPronunciationDetail]
    feedback_audio_path: Optional[str] = None       # path to TTS feedback file
    feedback_text: str
    comprehension_questions: Optional[List[str]] = None


def build_audio_agent():
    return create_agent(
        model="gpt-4o-mini",
        tools=[
            load_question_audio,
            transcribe_audio,
            score_pronunciation,
            generate_speech,
            generate_comprehension_passages,
        ],
        system_prompt=AUDIO_SYSTEM_PROMPT,
        response_format=AudioEvaluationOutput,
    )


async def run_audio_agent(
    question_text: str,
    reference_answer: str,
    user_audio_path: str,
    question_id: str,
    exercise_type: str = "pronunciation",
) -> dict:
    """
    Orchestrate a full listening evaluation cycle.

    Args:
        question_text:    The question or prompt read aloud to the user.
        reference_answer: The expected spoken answer.
        user_audio_path:  Path to the user's recorded audio (inside audios/).
        question_id:      Unique ID for naming output files.
        exercise_type:    'pronunciation' | 'comprehension'
    """
    print(f"AUDIO AGENT: Starting evaluation for question_id={question_id}")
    agent = build_audio_agent()

    user_message = (
        f"Exercise type: {exercise_type}\n"
        f"Question: {question_text}\n"
        f"Reference answer: {reference_answer}\n"
        f"User audio file: {user_audio_path}\n"
        f"Question ID: {question_id}\n"
        "Please evaluate the user's pronunciation and generate feedback."
    )

    result = agent.invoke({"messages": [{"role": "user", "content": user_message}]})
    structured: AudioEvaluationOutput = result["structured_response"]

    print(f"AUDIO AGENT: Evaluation complete — score={structured.score}")
    return structured.model_dump()
```

### `app/routes/audio_routes.py`

```python
# app/routes/audio_routes.py
from fastapi import APIRouter
from fastapi.responses import FileResponse
from pydantic import BaseModel
from pathlib import Path
from ..services.audio_service import evaluate_audio

router = APIRouter(prefix="/api/v1/audio", tags=["audio"])


class EvaluateRequest(BaseModel):
    question_text: str
    reference_answer: str
    user_audio_path: str      # relative path inside audios/
    question_id: str
    exercise_type: str = "pronunciation"


@router.post("/evaluate")
async def evaluate(req: EvaluateRequest):
    result = await evaluate_audio(
        question_text=req.question_text,
        reference_answer=req.reference_answer,
        user_audio_path=req.user_audio_path,
        question_id=req.question_id,
        exercise_type=req.exercise_type,
    )
    return result


@router.get("/file/{filename}")
async def serve_audio(filename: str):
    """Serve an audio file from the audios/ folder (for React Native playback)."""
    path = Path("audios") / filename
    if not path.exists():
        from fastapi import HTTPException
        raise HTTPException(status_code=404, detail="Audio file not found")
    return FileResponse(path, media_type="audio/mpeg")
```

## Step 8 — View hierarchical traces in LangSmith

1. Open LangSmith → your project.
2. Find the trace named after your `LANGSMITH_PROJECT`.
3. The trace tree will look like:

```
run_audio_agent                          ← top-level agent span
  └── AgentExecutor
        ├── load_question_audio          ← TTS for question
        │     └── generate_speech
        ├── transcribe_audio             ← Whisper transcription
        ├── score_pronunciation          ← rapidfuzz scoring
        ├── generate_speech              ← TTS feedback audio
        └── (generate_comprehension_passages)   ← if type=comprehension
```

4. Click any span to see inputs, outputs, token usage, and latency.
5. Use the **Compare** view to diff two evaluation runs side-by-side.

## Step 9 — Answer validation for comprehension exercises

When `exercise_type == "comprehension"`, the agent calls `generate_comprehension_passages` to produce a reading passage and questions. To also validate a user's written answer, reuse the existing `validate_question_quality` tool already in the agent or add a lightweight `validate_answer` tool:

```python
# app/tools/answer_validation_tool.py
from langchain_openai import ChatOpenAI
from langsmith import traceable

_llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

@traceable(name="validate_answer")
def validate_answer(question: str, correct_answer: str, user_answer: str) -> dict:
    """
    Check whether the user's answer is semantically correct.

    Args:
        question (str): The comprehension question.
        correct_answer (str): The model answer.
        user_answer (str): The student's submitted answer.

    Returns:
        dict with keys: is_correct (bool), explanation (str), similarity_score (int)
    """
    prompt = (
        f"Question: {question}\n"
        f"Correct answer: {correct_answer}\n"
        f"Student answer: {user_answer}\n\n"
        "Is the student's answer correct or substantially equivalent? "
        "Reply with JSON: {\"is_correct\": true/false, \"explanation\": \"...\", \"similarity_score\": 0-100}"
    )
    response = _llm.invoke(prompt)
    import json
    return json.loads(response.content)
```

Add `validate_answer` to the tools list in `build_audio_agent()`.
